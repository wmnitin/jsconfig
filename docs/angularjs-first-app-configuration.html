AngularJS has been included in above code using CDN, I have made some fields as bold for better understanding.
That’s all, you don’t need any extra ordinary software to run this code, just open this html file in your browser and enter USA in Name field & Hello USA will be displayed automatically below it.  I know I am telling very basic things but I have to explain these all for beginners, please don’t be bore. 😉
Now let me explain you some basics of AngularJS terminologies that will help you in future.

AngularJS directives:

All Html attributes starting with ng– are called AngularJS directives. In our above example, we have used two directives ng-app & ng-model. There are many directives like this but I am not going to list all these here right now.

Some basic AngularJS directives:

ng-app : This is mandatory before using any AngularJS application otherwise your code will not work. This tells our web page that we are using AngularJS application having defined name.
ng-model : It is used to bind any data entered by user to HTML, after getting the data into ng-model we can use it throughout our AngularJS application.
ng-bind : It is used to display the data which is already in ng-model. You won’t be using it frequently because same functionality can be achieved by AngularJS expression {{…}}

AngularJS expressions:

Whenever we want to display the data of AngularJS model then we use Expressions. In above example {{anyname}} is your expression that is displaying data of model ng-model=”anyname” . Alternatively you can also use AngularJS directive ng-bind=”anyname” .

AngularJS controllers:

In this example we did not use any controller, but most of the times you will work only on controllers only so we can’t miss this topic. Check out my another tutorial that is only focusing on controllers : Working with AngularJS Controllers

Working of AngularJS Application :

Earlier to perform any dynamic function in html static page, we used to get help of simple Javascript, Jquery etc, AngularJS is just a framework of Javascript. AngularJS can extend HTML attributes to different level. AngularJS can bind & fetch any data from HTML attributes.
So, most of the time you will use AngularJS to fetch data from server using ajax calls to REST API & display them to HTML pages by binding into HTML attributes. Or in reverse order, Fetch data from html forms & store them into server by sending data to REST API through ajax call.
Hence by using Ajax calls to server every time whenever we need data or post data , we don’t require to reload our page. Apart from this AngularJS proved full configurations to work on url routes & templates. So, we can say AngularJS is perfect for single page applications.
