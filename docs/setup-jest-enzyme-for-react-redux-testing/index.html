You must be knowing some testing tools like jasmine, karma etc if you have been doing unit test & integration test in JavaScript applications. But once you move into ReactJS application development, you will be needing tools like jest & enzyme to test your application fully.
JEST is developed by Facebook & you can use it for any JavaScript project, so it is not necessary whether your project is on ReactJS or any other framework. If your project is on RectJS, you will be needing another tool that is enzyme. Enzyme provides extra features like testing of Reactjs component, rendering, mounting, updating etc.
In this tutorial, we will provide some steps by which you can configure these tool with better architecture for your javascript project. Let’s get started.

Install & Configure Jest

1
npm install --save-dev jest
Yes that’s all, you don’t need anything extra to run your first test case. you are done. But if your project is big enough, you will have to write jest configuration to make it run according to your need.
No need to create any extra config file, you can configure jest in your package.json file too. Now, let’s see how.

This is the basic config that can get you started with bigger projects. All you have to do is to write you unit cases in any __test__ folder obviously outside node_modules folder or you can write your test case in any file having name like this filename.test.js (test keyword is required). Most of the configuration is pretty understandable. Feel free to read more configuration options from here.

Install & Configure Enzyme

Want to test lifecycles & render of Reactjs Component ? lets configure Enzyme for you project now.
1
npm i --save-dev enzyme enzyme-adapter-react-16
Now tell you jest config to use enzyme config. Go back to your package.json & add one more line that is :
1
2
3
"setupFiles": [
  "./enzime.config.js"
]
Now create enzime.config.js in your root directory of your project & write enzyme configuration files for ReactJS v16 like this:
1
2
3
4
import { configure } from 'enzyme'; //configure enzyme
import Adapter from 'enzyme-adapter-react-16';//adapter for React16
 
configure({ adapter: new Adapter() }); //configured
That’s all, now your configuration for both jest & enzyme has been done. Now you can proceed to writing unit test cases.

Recommended Test cases architecture

Now you will be writing unit cases for each & every component of your project. We suggest ton write test case file in the same directory in which your component is placed. It will help you to manage your test cases efficiently.
For example: Suppose Component.js is placed somewhere inside your project, It’s better to create test file like Component.test.js in the same directory. Just make one file for every component you want to test.

import React from 'react';
import { shallow } from 'enzyme';
import Requirement from './Requirement';
 
describe('<Requirement />', () => {
    it('initial state of data should be null', () => {
        const wrapper = shallow(<Requirement />);
        expect(wrapper.state().data).toEqual(null);
    });
})
