Without learning usage of Custom AngularJS directives., it is incomplete to learn AngularJS, you can create your own directives just like ng-bind , ng-app etc. Creating AngularJS custom directives are very easy & in this post we will talk about creation, basic uses, properties, using scope in custom directives, uses of compile & link in custom directives.

Introduction to Custom Directives :

There are four types of directives that you can create in AngularJS, I will list & explain each of them with help of example instead of boring theory. These are basically DOM structure of HTML.
  1. Elements : <jsconfig> jsconfig>  This is a basic element & you can create any element directive of your choice & AngularJS will do his work on them. Code : E
  2. Attribute : <div  ng-anything></div> Here ng-anything is your attribute & you can provide any name to your custom directive of attribute. Code : A
  3. Class : <div class=”jsconfig“></div> Custom directives can be created for class name, just define any class name & change code of that class through AngularJS directive. Code : C
  4. Comment: <!–jsconfig–> Simple, use any name inside comment section of HTML & you can modify its data by creating its custom directive. Code : M
Remember the code I have written at end of statement of every directives, we will be using the same code to restrict our application to reflect changes only on specified AngularJS custom directives.

Properties of AngularJS custom directives

Lets understand the basic properties of custom directives, some of them I have used in above example. Lets understand all to create better directives.
template : Using this , you can write any HTML code that you want to reflect in place of your custom directive
templateUrl : You can use this to include any html template, instead of using template you can also use templateUrl . It is good if you want to include long html code.
restrict = ‘E’   is the code of element, you can include it in to restrict your directive to display only in place of element. I have already mentioned all four codes above.
replace = ‘true’ I won’t explain you this because this property has been depreciated & won’t be available in next versions of AngularJS, I will suggest you to not use this.
scope = {
  data : “=”
}
Above code is important if you want to display any scope data, I think I should give you one example using scope.

Compile & Link function :

After learning basics properties of AngularJS custom directives, Its time for compile & link functions. In short, remember compile function will run only one time & it performs basic operations. Basic structure is like this, you can use compile to get element of html & do some custom stuffs.

.directive('jsconfig',function(){
   var directive={};
   directive.restric='E';
   directive.template="&lt;h2&gt;Welcome to JsConfig {{data}}&lt;/h2&gt;";
   directive.scope={ data: "="};
   directive.<strong>compile</strong> = function(element,attr){
      element.css('color','red')
   }
   return directive;
});
Link function is also similar to compile but it has one extra property that is scope & AngularJS calls it every time there is some changes in HTML code. You can use link function without compile. let’s have a look on example that I have created with usage of scope & element property of link.
