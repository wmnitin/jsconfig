In order to understand AngularJS application, you should know about working of some parameters/objects that you include in your controllers. In this tutorial I will talk about $scope , nested $scope , introduce you with $rootScope and will list out some differences between $scope and $rootScope.

$scope

In simple words, $scope holds the data of model that can be used in HTML views. $scope is a JavaScript object that is used to assign & access data between controller and view. With the help of $scope object we can assign any type of data to view either a string variable or any JSON data, but you can not access the scope of one controller into another, each time you will get new $scope object in different AngularJS controllers. Let’s try to understand this with help of an example.
Create two controllers “firstCtrl” and “secondCtrl” :

$rootScope

It solves the problem when you want to access or alter the data from any controller, unlike $scope. No need to define your $scope.customtext in every controller, just assign the value in one controller like $rootScope.customtext & it will be reflected in your all HTML views regardless of controllers. Lets have a look in this example :
1
2
3
4
5
6
var JsConfigApp = angular.module('app-name', [])
JsConfigApp.controller('firstCtrl', function($rootScope){
$rootScope.customtext="Hello I am in rootScope";
})
JsConfigApp.controller('secondCtrl', function(){
})
Create an HTML view now, by using both controllers:
1

<body ng-app="app-name">
<div ng-controller="firstCtrl">
<p> {{ customtext }} </p>
</div>
<div ng-controller="secondCtrl">
<p> {{ customtext }} </p>
</div>
</body>
Now the output will be “Hello I am in rootScope” two times. Look at below image that is clearly explaining the differences.
Now you have got the understanding of $scope & $rootScope , Its time to introduce a new term that is “Scope Inheritance” or “Controller Inheritance”. Yes right, you can inherit the data from one controller to another without using $rootScope. let’s have a look on next section nested $scope.

In my first example, I have used secondCtrl inside firstCtrl. Hence all the  $scope models will be automatically inherited to secondCtrl. If you do not assign any value in $scope.customtext in secondCtrl then also it will display the value after checking from parent controller.
So, the output of Example1 will be :
Hello I am in FirstCtrl
Hello I am in FirstCtrl
Now let’s have a look into second example, in second example I have assigned the value of customtext. Here also, data will be inherited but it will take the value of  child’s controllers. So, output will be:

Hello I am in secondCtrl
