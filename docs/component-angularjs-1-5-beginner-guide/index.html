Let’s try to analyse the code, we have created a component of <jsconfig> , similar way we used to create custom directive. we have also passed an attribute “name” in our HTML element <jsconfig> syntax. We are binding this attribute value inside component using ‘@’ , will let you know about bindings later in this post. When we run this example, our <jsconfig> element is replaced by the template value that is provided in component.

Bindings in AngularJS Component

There are four ways by which you can bind your data, let me describe them
‘@’ : This can be used when you are binding input string, using this does not need any binding from controller side although you can always change its value from controllerif it is not defined in view.
‘<‘ : This is used in case of one way data binding, when you want to read value from parent scope
‘=’ : Two way winding, you have to define it in controller too apart from template in order to use, data changed from controller will be reflected in view overriding initial view data.
‘&’ : Outputs are realised with this bindings, which function as callbacks to component events.

Using Controller in Component

Component is better replacement of custom directives with many powerful features,  but sometimes you need controllers to do certain things. Let’s see how you can use controller inside compoenent.

JsConfigApp.component('jsconfig',{
   template: "Hello {{$ctrl.name}}",
   bindings: {
   name: '@'
   },
   controller: function(){
      this.name='jsconfig is best site'
   }
})
You have successfully placed your controller inside component. I forgot to tell $ctrl is the default ControllerAS name, if you do not specify any it will take $ctrl. But, it does not mean you can not use ControllerAS , you can always use it & specify your custom name of controller.
Instead of writing whole controller in component, you can write your controller in old way too & give your controller name in place of function, look at below syntax for understanding:

templateUrl

You cal use templateUrl if your html code is too big, just replace your template with templateUrl and provide path of your file & you are good to go.

Inheriting data from parent component

There is a property name “require” which allows to fetch any data from parent component, this is useful when you are creating component of nested element, instead of creating any data again , just inherit it from parent. Look at syntax below:

JsConfigApp.component('jsconfig',{
   template: "AngularJS rocks",
   require{
       parent: '^parentComponenet'
   }
})
