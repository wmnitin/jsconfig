In this tutorial I am going to show you the usage of directive formControl in Angular2. Using standalone , I mean to use formControl without any form tag. I am writing this tutorial with respect to new RC5 version of Angular. The formControl directive is a part of ReactiveFormsModule, hence you have to import ReactiveFormsModule in your app module before using formControl like this :

import { BrowserModule } from '@angular/platform-browser';

  imports:      [ BrowserModule, ReactiveFormsModule ],
  declarations: [ AppComponent ],
  bootstrap:    [ AppComponent ]
})
export class AppModule { }
As you are done with configuration, let’s move to next step that is creating an html input. See, in this tutorial I am not explaining you any kind of forms like template driven or model driven, I just want to show you the usage of formControl without any form. For model driven form please refer another tutorial or refer tutorial for template driven forms.
Lets make an input having one formControl attribute. We will fetch its value & perform some validation too with this input using formControl in Angular2.

      <input type="email" class="form-control" name="email" placeholder="Email" [formControl]="emailControl" (change)="performLogin()" />
      <p style="color:Red" *ngIf="emailControl.invalid" >Email is required</p>
    </div>
</div>
As you can see above, I am not using any form tag. I have used [formControl]=”emailControl” that will associate with FormControl instance in your component. Apart from this I have used (change) event that will be useful to fetch this input field data. Let’s move to its component now.

import { Component } from '@angular/core';
import { FormControl } from '@angular/forms';
 
@Component ({
selector:'my-demo',
templateUrl:'./app/users/mydemo.component.html'
})
export class MydemoComponent{
constructor(){
};
emailControl = new FormControl();
performLogin(){
console.log("email entered is "+this.emailControl.value)
};
}
As you can see, I have imported the module FormControl first, the attached its instance with emailControl. This is required to tell Angular2 that you are using FormControl. When emailControl value will get change it will call a function performLogin(). In performLogin(), I have written a logic to print input value. So, easy till now.

import { Component } from '@angular/core';
import { FormControl, Validators } from '@angular/forms';
 
@Component ({
selector:'my-demo',
templateUrl:'./app/users/mydemo.component.html'
})
export class UserLoginComponent{
constructor(){
};
emailControl = new FormControl('', Validators.required);
performLogin(){
console.log("email entered is "+this.emailControl.value)
};
}
If you see in first html code, I have added a paragraph tag that will only be visible if your emailControl value is valid. Hence, it made your validation task very easy.

import { Component } from '@angular/core';
import { FormControl, Validators } from '@angular/forms';
 
@Component ({
selector:'my-demo',
templateUrl:'./app/users/mydemo.component.html'
})
export class UserLoginComponent{
constructor(){
};
 
emailControl = new FormControl('', Validators.required);
 
ngOnInit(){
console.log("working")
this.emailControl.valueChanges.subscribe(
value=>console.log(value)
)
}
}
