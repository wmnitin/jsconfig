Webpack version 4 is now fully stable & can be used in any type of projects from small to big. We have recently converted our big project from webpack 3 to 4 & we faced some outdated plugin challenges but finally it was worth to upgrade. I assume you already know the features of webpack 4, so I am going to be straight forward with the installation & setup.

Webpack 4 Installation / Upgrade :

Go into your project directory & run this webpack cli installation command using npm
1
npm install --save-dev webpack-cli
This will install webpack version 4 latest version. Make sure to check version in your package.json file.

Webpack development & production file setup :

As you all know webpack4 support –mode flag which is very useful if you want to keep same config file for development and production.
But if your project is big & you want to keep development build in separate directory & production build in separate, you would end up creating separate config files for both environment as you used to do in earlier versions of webpack.
Don’t worry you can create two files lets save : webpack.dev.js & webpack.prod.js
In your package.json add these two scripts for running these config:
1
2
"dev": "webpack --mode development --progress --profile --config webpack/webpack.dev.js",
"prod": "webpack --mode production --progress --profile --config webpack/webpack.prod.js"

Suppose at any time inside your config you want to read the –mode flag value, you can do it easily like this :
1
2
3
module.exports = (env, argv) => {
console.log(argv.mode); //Output will be "production" or "development"
}

No need to add plugins like Uglify etc for production, because webpack4 will automatically include production related plugins if you use –mode production.

Code splitting & Chunking in Webpack 4:

CommonsChunkPlugin that is widely used in earlier versions of webpack has been removed, so this may be the major change that you will have to remember. So instead of this plugin, Webpack came up with enhanced & advanced optimization option that is splitChunks.
Webpack4 has good documentation of this plugin, you can refer it from here.
I am going to show you, how we have used this plugin in our code.

        optimization: {
            splitChunks: {
                chunks: "async",
                automaticNameDelimiter: '-',
                name: true,
                cacheGroups: {
                    vendors: {
                        test: /[\\/]node_modules[\\/]/,
                        priority: -10
                    }
                }
            }
        }

I am going to be very brief in explanation of above code. Suppose I have a JS file name abc.js that is using some tooltip plugin from node_modules folder. Think, how chunks will be created ? Chunks will be like this
abc.adsf342dvdbsgsg.js
vendors-abc.cafsgshsh231bd.js
Yes it will create separate vendor file for every code that is coming from node_modules folder. You can use this in many ways & we have already started loving this feature.
And for code spliting it is same as webpack 3, use dynamic import.

SASS , CSS, FONT Loader in webpack 4

If you have used Webpack3 that you must be knowing about this plugin ExtractTextPlugin, yes we have used this plugin for sass loader etc tasks. Unfortunately this plugin will not work in Webpack 4, so use MiniCssExtractPlugin. ExtractTextPlugin has stopped providing support for newer version of webpack, but MiniCssExtractPlugin is superb  replacement & we didn’t face any issue with this till now.
Usage is simple & somehow similar to extract text plugin. Here is our config :

const MiniCssExtractPlugin = require("mini-css-extract-plugin");
module.exports = (env, argv) => {
     return module: {
            rules: [{
                test: /\.scss$/,
                use: [
                    argv.mode == 'development' ? 'style-loader' : MiniCssExtractPlugin.loader,
                    "css-loader",
                    {
                        loader: "postcss-loader", options: {
                            plugins: () => [require('autoprefixer')]
                        }
                    },
                    "sass-loader"]
            }, {
                test: /\.font\.js/,
                use: [MiniCssExtractPlugin.loader,
                    'css-loader',
                    'webfonts-loader'
                ]
            }]
        },
        plugins: [
            new MiniCssExtractPlugin({
                filename: 'c/[name].[chunkhash].css',
                chunkFilename: 'c/[name].[chunkhash].css'
            })
        ]
    }
};

Above config is very easy to ready if you are familiar with webpack.
